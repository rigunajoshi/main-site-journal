# main-site-journal

## 项目简介

`main-site-journal` 是一个用于归档和发布多个独立 HTML 页面的仓库。  
本仓库不针对任何特定域名或网站，主要用于保存和展示各类静态页面内容，便于版本管理和历史追溯。

## 目录结构

```
main-site-journal/
├── pages/          # 存放所有 HTML 页面文件
├── assets/         # 页面引用的资源文件（图片、样式、脚本等）
├── archive/        # 历史版本或不再更新的页面归档
└── README.md       # 本文件
```

- `pages/`：每个 HTML 文件代表一个独立页面，文件名应反映页面主题或用途。
- `assets/`：按需组织子目录，如 `images/`、`styles/`、`scripts/`。
- `archive/`：存放已归档的旧版本页面，保留原始结构与内容。

## 页面归档说明

- 所有页面均为纯静态 HTML，不依赖后端服务。
- 每个页面应保持自包含，资源引用使用相对路径。
- 如需更新页面内容，请直接修改 `pages/` 下对应文件，并在提交信息中注明变更摘要。
- 被替代的旧版本可移入 `archive/` 目录，文件名建议添加日期或版本后缀。

## 维护说明

- 欢迎提交 Pull Request 添加新页面或改进现有内容。
- 提交前请确保页面在主流浏览器中正常渲染。
- 请勿在页面中包含任何形式的推广、链接交换或商业诱导内容。
- 仓库维护者会定期审核提交，保持内容整洁与结构清晰。
